### 5446\. 三次操作后最大值与最小值的最小差

这题刚开始想到的是，先排序然后最大的变最小的或者最小的变最大，但是结果仍然不对，这2种方案适合在数组超过5个数的时候还管用，在做题时卡在了5个数的数组，心想不可能暴力求出来吧。最后发现原来还缺了2种方案。

func minDifference(nums \[\]int) int {
   if len(nums) < 4 {return 0}
   sort.Ints(nums)
   //依次对应四种方案: 去除最小3个/去除最小2个和最大1个/去除最小1个和最大2个/去除最大3个
   return min(min(nums\[len(nums)-1\]-nums\[3\], nums\[len(nums)-2\]-nums\[2\]), min(nums\[len(nums)-3\]-nums\[1\], nums\[len(nums)-4\]-nums\[0\]))
}

func min(a, b int) int {
   if a > b{
      return b
   }
   return a
}

### 5447\. 石子游戏 IV

这道题很明显是博弈DP，就不多说了主要是理解性的东西，请看下面的代码注释。

func winnerSquareGame(n int) bool {
   dp := make(\[\]bool, n+1)
   for i := 0 ; i <= n ; i++ {
      for j := 1 ; j\*j+i <= n ; j++ {//取几个
         if(!dp\[i\]) {//如果第i个是Bob取,那么Alice可以取到j\*j+i为true
            dp\[j\*j+i\] = true
         }
      }
   }
   return dp\[n\] //true表示这局是Alice,false表示这局是Bob
}
